## pr-review

Review pull requests effectively.

### What to check
- Does code match PR description?
- Tests for new functionality?
- Error handling appropriate?
- Security concerns?
- Performance acceptable?
- Breaking changes?
- Code style consistent?

### Review types
- Code review: logic, patterns, bugs
- Security review: vulnerabilities
- Design review: architecture
- UX review: user impact
